
### 1. Please list the hierarchy of DNS databases.
DNS 採用分散式、階層式的資料庫設計，主要分為以下三層：
* 根名字伺服器 (Root name servers)：位於階層最頂端，負責引導查詢至對應的 TLD 伺服器。
* 頂層網域伺服器 (Top-level domain (TLD) name servers)：負責管理如 .com, .org, .edu 或國家代碼如 .tw, .in 等網域。
* 權威名字伺服器 (Authoritative name servers)：由各組織自行維護，提供該組織內主機（如 www.nchu.edu.tw）的正式 IP 位址對應。

### 2. Why does DNS use distributed databases?
DNS 不採用單一集中式伺服器，主要是為了避免以下問題 ：
* 單點故障 (Single point of failure)：若中央伺服器故障，全球網路將癱瘓。
* 流量負擔 (Traffic volume)：全球查詢請求量極大，單一台伺服器無法負荷。
* 距離與延遲 (Distant centralized database)：查詢請求若需跨海傳輸至遠端伺服器，會造成嚴重的反應延遲。
* 維護與擴充性 (Maintenance and scalability)：分散式架構便於各網域自行管理與擴充紀錄。

### 3. Please describe the procedures of iterated and recursive query for a local name server.
當本地名字伺服器 (Local Name Server) 收到查詢請求時 ：
* 遞迴查詢 (Recursive Query)：本地伺服器代為向外查詢，它會詢問下一層伺服器，下一層再詢問下下層，直到獲得答案後回傳給用戶端。用戶端只需等待最終結果。
* 疊代查詢 (Iterated Query)：本地伺服器詢問根伺服器，根伺服器不直接給答案，而是回傳「下一個該問誰」的位址（如 TLD 伺服器位址），本地伺服器再親自去詢問該位址，反覆此過程直到找到權威伺服器。

### 4. Please list and describe four typical types of DNS resource records.
DNS 紀錄包含類型 (Type) 欄位，常見的有：
* Type A (Address)：將主機名稱映射至其 IPv4 位址（例如 www.nyu.edu 映射至 216.165.47.12）。
* Type NS (Name Server)：指定負責該網域的權威名字伺服器主機名稱 。
* Type CNAME (Canonical Name)：將別名指向其真正的規範名稱（如 www.google.com.tw 指向 www.google.com）。
* Type MX (Mail Exchange)：指定負責接收發往該網域郵件的郵件伺服器名稱 。

### 5. How does dynamic DNS avoid caching old IP addresses?
為了防止用戶端讀取到已變更但仍在快取中的舊 IP，DNS 使用了以下機制：
* 存留時間 (TTL, Time to Live)：每條 DNS 紀錄都有 TTL 值，規定該紀錄在快取中可存放的時間。
* 超時清除：一旦快取時間超過 TTL，紀錄就會被自動移除，下次查詢必須重新詢問伺服器獲取最新 IP 。

### 6. Please describe the main ideas of BT to accelerate file distribution.
BT 透過 P2P 架構 大幅提高分發效率，主要概念包括：
* 切片分發 (Chunks)：將大檔案切成許多 256KB 的小塊，讓 peer 之間可以互相交換已下載的部分。
* 最稀有優先 (Rarest First)：優先下載群組中最少人擁有的片段，確保片段不會在網路中消失。
* Tit-for-tat (以牙還牙)：優先將資料傳給那些也正在積極傳送資料給自己的 peer，以提高下載速率。

### 7. How does BT deal with the problem that a peer selfishly leaves after finishing downloading?
BT 使用 激勵機制 來減少「搭便車」行為：
* 貢獻度獎勵：只有持續上傳資料的 peer 才能從其他 peer 那裡獲得較高的下載配額。
* 種子 (Seeds)：鼓勵下載完成的 peer 繼續留在群組中擔任「種子」，僅供上傳而不下載，回饋社群。

### 8. Please describe the operations of DASH.
DASH 允許視訊根據網路狀況動態調整畫質：
Server-Side Operations: 將原始視訊切割成數秒的小段，並針對不同頻寬壓製多種品質等級（如 4K, 1080p, 720p），並提供一個 告示檔案 (Manifest file) 描述各版本位置。
Client-Side Operations: 用戶端會持續測量下載速率，並根據當前網速向伺服器請求對應品質的視訊片段。

### 9. Please describe the functions of CDNs.

CDN 透過在地理上分散的伺服器節點（Edge Servers）緩存內容，達到以下功能：
* 降低延遲：讓使用者從地理位置最近的伺服器獲取資料。
* 分擔負荷：減少原始伺服器（Origin Server）的頻寬壓力。
* 提高可靠性：當部分節點故障時，流量可自動轉向其他健康節點。

### 10. Please explain which operations of CDNs rely on DNS.
CDN 的「重導向 (Redirection)」機制高度依賴 DNS：
* 當使用者查詢主機名稱時，CDN 的權威伺服器會分析使用者的 來源 IP 位址。
* 透過 DNS 回傳一個特定的 CNAME 或 IP，將查詢引導至目前距離使用者最近、負載最輕的 邊緣伺服器 (Edge Server) 節點。
