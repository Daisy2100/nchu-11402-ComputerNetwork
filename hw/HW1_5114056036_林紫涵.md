### 4. What is the equation for calculating transmission delay?
* 假設傳送距離為d, 傳輸的速率為s, 那麼傳播延遲= d/s

### 6. What is “store and forward”?
* 是指路由器在將封包傳輸到下一個鏈路之前，必須先完整接收到整個封包。
* 在封包交換的過程中，當封包到達路由器時，路由器會將接收到的位元暫存在內部。直到該封包的最後一個位元都抵達且儲存後，路由器才會開始將該封包轉發（傳輸）到通往目的地的下一個鏈路上。

### 7. Please list the consequences of network congestion for packet switching.
* 網路壅塞會導致 Queueing delay 以及 Packet loss。

### 8. Please list the pros and cons of circuit switching and packet switching.
*   電路交換：
    *   優點：提供專用的網路資源（不共用），能保證類似電路的穩定傳輸效能。
    *   缺點：如果通話或連線中未充分使用該電路區段，資源就會閒置，造成浪費。
*   封包交換：
    *   優點：允許多個主機與應用程式彈性共用網路資源與鏈路（基於資源排隊的特性）。
    *   缺點：當流量到達速率大於鏈路傳輸速率時，會發生佇列等待，導致不可預測的延遲，嚴重時甚至會發生封包遺失。

### 9. Please list and describe the definitions of four sources of packet delay.
1.  節點處理延遲：路由器檢查封包標頭與決定將封包導向何處等處理所需的時間。
2.  佇列延遲：封包在路由器緩衝區中等待被傳輸到輸出鏈路的時間。
3.  傳輸延遲：將封包所有位元推入傳輸鏈路所需的時間。
4.  傳播延遲：訊號在實體介質中從一個節點傳播到下一個節點所需的時間。

### 10. Please list Internet five-layer protocol stack.
*   應用層：包含 HTTP, SMTP, QUIC, DASH, RTP 等協定。
*   傳輸層：包含 TCP, UDP 協定。
*   網路層：是網際網路的「細腰」，由單一的 IP 協定主導。
*   鏈結層：包含 Ethernet, WiFi, Bluetooth, PPP 等。
*   實體層：實體涉弊。


### 11. How and why packet headers are stacked?
* 透過「封裝（Encapsulation）」堆疊。
* 當資料從應用層往下傳遞到實體層時，每一層的協定都會在上一層交下來的資料前面，加上該層專屬的控制資訊（Header）。
* 這樣做的原因是為了讓每一層都能各自處理獨立的功能（例如：傳輸層負責端對端可靠性，網路層負責路由尋址），實現分層架構；而在接收端則會依序「解封裝」剝離對應的標頭資訊。

### 12. Please list the four network-service requirements of network applications.
1. 資料完整性：某些應用（如檔案傳輸、電子郵件）要求 100% 的資料可靠性，不容許遺失。
2. 時效性：即時應用（如網路電話、連線遊戲）對延遲非常敏感，需要保證極低的延遲。
3. 傳輸量：某些應用（如多媒體影音串流）需要保證最低的頻寬才能順暢運作。
4. 安全性：需要機密性（加密）、確保資料不被竄改。

### 13. Please compare the client-server and peer-to-peer architectures for network applications.
*   client-server：有一台永遠開啟且具備固定 IP 的伺服器負責服務眾多客戶端。客戶端彼此之間通常不會直接通訊。優點是易於集中管理與維護，缺點是伺服器可能會單點失效。
*   peer-to-peer：沒有集中式的伺服器，任意的主機之間可以直接通訊並互相提供服務。優點是具有極佳的擴展性，因為每個節點既是消費者也是提供者；缺點是難以管理、IP 變動頻繁且安全性較難控管。

### 14. Please describe the difference between non-persistent and persistent HTTP.
*   Non-persistent HTTP：每個 TCP 連線只能傳輸「一個」請求/回應物件。傳送完該物件後，TCP 連線就會關閉。若要下載多個物件，需為每一個物件重新建立 TCP 連線，導致較大的延遲與系統開銷。
*   Persistent HTTP：允許在同一個 TCP 連線上接續傳送「多個」請求與回應。這樣可大幅減少重複進行 TCP 三方交握的開銷，加快網頁整體載入速度。

### 16. Please describe the components of HTTP cookies.
1. HTTP 回應訊息中的 cookie 標頭行（例如 `Set-Cookie:`）：伺服器用來指派 ID 給客戶端。
2. HTTP 請求訊息中的 cookie 標頭行（例如 `Cookie:`）：客戶端在後續請求中帶上此 ID 讓伺服器辨識。
3. 客戶端（瀏覽器）保留的 Cookie 檔案：由使用者的瀏覽器負責管理與儲存。
4. 網站後端資料庫：伺服器利用接收到的 Cookie ID 到資料庫中尋找該使用者的狀態或歷史紀錄。

### 17. Please list the benefits of web cache.
* 降低客戶端請求的回應時間
* 減少機構內部存取鏈路的流量
* 減輕原始伺服器的負載

### 18. Please describe how HTTP/2 mitigates HOL blocking.
* Multiplexing & * Framing
* 在 HTTP/1.1 中，如果一個大物件正在傳輸，後面的小物件請求必須等待（即產生 HTTP 層級的 HOL 阻塞）。
* HTTP/2 將 HTTP 訊息分割成許多小型的「Frames」，並在同一個 TCP 連線上將不同請求的回應Frames交錯傳輸。到達接收端後再根據標頭重新組裝。這樣一來，大型物件的傳輸就不會完全阻塞住後方小型物件的傳輸。
